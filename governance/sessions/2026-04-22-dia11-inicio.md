## Riesgos conocidos

1. **Secret keys ausentes:** si no tenemos las credenciales actuales, necesitamos regenerar
2. **Socket Mode conexión:** primera vez puede tomar 30-60s conectar
3. **ECS task restart loop:** si secret está mal, task reinicia constantemente
4. **Costo inicial:** primeras horas pueden generar ~$0.01-0.02 hasta optimizar

## Rollback plan

### Runtime Issue Detectado y Resuelto

**Problema inicial:**
- Atlas respondía en Slack con: "Tuve un error procesando tu solicitud"
- CloudWatch logs mostraban: `httpcore.LocalProtocolError: Illegal header value b'sk-ant-api03-...\nsk-ant-api03-...'`

**Diagnóstico:**
Secret `nova/atlas/felirni/config` tenía `anthropic_api_key` duplicado con separador `\n`:
```
"anthropic_api_key": "sk-ant-api03-...\nsk-ant-api03-..."
```

**Fix aplicado:**
```bash
# 1. Recuperar secret
aws secretsmanager get-secret-value \
  --secret-id nova/atlas/felirni/config \
  --region us-east-1 \
  --query SecretString --output text > /tmp/secret-original.json

# 2. Fix: remover duplicado
cat /tmp/secret-original.json | \
  jq '.anthropic_api_key = (.anthropic_api_key | split("\n")[0])' \
  > /tmp/secret-fixed.json

# 3. Update secret
aws secretsmanager update-secret \
  --secret-id nova/atlas/felirni/config \
  --secret-string file:///tmp/secret-fixed.json \
  --region us-east-1
```

**ECS task restart:**
```bash
# Stop old task (forzar restart)
aws ecs stop-task \
  --cluster nova-atlas-cluster-prod \
  --task 13f43cf7b2064f0495ff79d7fe61adaf \
  --reason "Secret fix applied" \
  --region us-east-1

# New task auto-started por ECS service
# Task ID: 2dd54478ef6f42e9a8d67d8e2a0c4d19
```
